# docs/index.html 多言語対応（i18n）設計

## 概要

RemoCoder のホームページ（`docs/index.html`）を日本語・英語の2言語に対応する。
ビルドシステムを追加せず、静的HTMLとして管理しやすい構成を採用する。

## 対応言語

- 日本語（ja）— プライマリ
- 英語（en）

## ファイル構成

```
docs/
├── index.html              # リダイレクタ（navigator.language で /ja/ or /en/ に振り分け）
├── style.css               # 共通CSS（現行 index.html の <style> を外部化）
├── ja/
│   ├── index.html          # 日本語版（現行から移動・CSS外部化）
│   └── privacy-policy.html # 日本語版プライバシーポリシー（現行から移動）
└── en/
    ├── index.html          # 英語版（新規作成）
    └── privacy-policy.html # 英語版プライバシーポリシー（新規作成）
```

## 実装詳細

### 1. リダイレクタ (`docs/index.html`)

- `navigator.language` が `ja` で始まれば `./ja/` に `location.replace()` でリダイレクト
- それ以外は `./en/` にリダイレクト
- JS無効環境は `<noscript><meta http-equiv="refresh" content="0; url=./ja/">` で日本語フォールバック
- `location.replace()` を使うことでブラウザ履歴にリダイレクタが残らない

### 2. 共通CSS (`docs/style.css`)

- 現行 `index.html` の `<style>` ブロックをそのまま外部ファイルに切り出す
- 各言語の `index.html` から `<link rel="stylesheet" href="../style.css">` で参照
- 言語スイッチャー用CSSを追加:

```css
.lang-switch { display: flex; gap: 4px; align-items: center; }
.lang-switch a[aria-current="true"] { color: var(--accent); }
.lang-switch span { color: var(--text-dim); }
```

### 3. 各言語ページの共通変更

- `<html lang="ja">` または `<html lang="en">`
- `hreflang` メタタグ:
  ```html
  <link rel="alternate" hreflang="ja" href="../ja/">
  <link rel="alternate" hreflang="en" href="../en/">
  <link rel="alternate" hreflang="x-default" href="../en/">
  ```
- ナビに言語スイッチャーを追加（現在の言語は `aria-current="true"` でハイライト）:
  ```html
  <li class="lang-switch">
    <a href="../ja/" aria-current="true">JA</a>
    <span>/</span>
    <a href="../en/">EN</a>
  </li>
  ```
- フッターのプライバシーポリシーリンクを `privacy-policy.html`（相対パス）に更新

### 4. 翻訳スコープ

以下のすべてのテキストを英訳する:

- `<title>` および OGP/Twitter メタタグ
- ナビゲーションリンクラベル
- 各セクションの見出し・本文・説明文
- ボタンラベル・ラベルテキスト
- ターミナルモックアップ内の日本語テキスト（`接続済み` など）
- FAQの質問・回答
- フッターのコピーライト

## SEO

- 各ページに `hreflang` タグを追加
- `x-default` は英語ページを指定
- リダイレクタ（ルート）には本文を持たせないためSEOインパクトは最小
